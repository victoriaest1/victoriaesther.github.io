The 3D tilt interaction

The hero card tilts on mouse move using CSS perspective() and a JS event listener — no library, under 15 lines.

tilt.js
magnetic.js
transition.js
// 3D tilt effect — hero card on mousemove const card = document.getElementById('hero-tilt-card');

card.addEventListener('mousemove', e => { const rect = card.getBoundingClientRect(); const x = (e.clientX - rect.left) / rect.width - 0.5; const y = (e.clientY - rect.top) / rect.height - 0.5; card.style.transform = `perspective(700px) rotateY(${x * 14}deg) rotateX(${-y * 14}deg) scale(1.03)`; card.style.boxShadow = `${-x * 24}px ${-y * 18}px 48px rgba(212,83,126,0.13)`; });

card.addEventListener('mouseleave', () => { card.style.transform = ''; // spring-back via CSS transition card.style.boxShadow = ''; });
Technical challenges

Problems I had to solve

Problem
Gradient text + CSS transform conflict
Applying -webkit-text-fill-color: transparent on gradient text caused the 3D tilt to show a black background instead of the page on some browsers.
→ Fixed with isolate + z-index stacking context
Solution
Isolated each gradient text element
Wrapping gradient text in a container with isolation: isolate and keeping the tilt transform on the parent card resolved the compositing issue cleanly.
→ Works across Safari, Chrome, Firefox
Problem
Scroll reveal re-triggering on filter
When filtering project cards, newly rendered cards had the .reveal class but were never observed by the IntersectionObserver since it was set up at init.
→ Observer needed re-registration
Solution
Observe after every renderCards() call
After each renderCards() call, re-queried for #cards-grid .reveal and registered each with observer.observe(). Immediately triggers visible ones already in viewport.
→ Smooth filter transitions every time
