The Challenge

Raw data is useless without a lens

Finance teams export CSVs constantly. Most of those files sit unopened because non-analysts can't make sense of them without pivot tables, SQL knowledge, or hours in Excel. The goal was to build something anyone could use to explore financial data instantly — no technical setup required.

One hard constraint from the start: no backend, no accounts, no servers. Data is personal — it should never leave the user's browser. Everything had to run entirely client-side.

The Solution

Drag, drop, explore

Users drag a CSV onto the page and immediately see category breakdowns, trend lines, and month-over-month comparisons — all rendered with D3.js in real time. Column headers are auto-detected, dates parsed automatically, and charts update instantly as users filter by date range or category.

Everything runs client-side. No data ever leaves the browser. The app is deployed as static files on GitHub Pages — no server costs, no maintenance, no downtime risk.

Reflections

What I'd do differently

D3.js has a steep learning curve — start with the data model. I jumped straight into the DOM manipulation and got lost. Starting with a clean data pipeline before touching rendering made the second half of the project twice as fast.
Date parsing edge cases are endless. Different locales, different separators, different column names for the same concept. Building a fuzzy date parser from day one would have saved three days of debugging.
